# Datasets for Many-Shot Residual Stream Geometry

This directory contains data generators and documentation for the research
project. Data files are NOT committed to git due to size. Use the generators
below to create data locally.

## Quick Start

```bash
# From the project root, using the project venv:
source .venv/bin/activate

# Generate all HMM datasets (Mess3, Z1R, RRXOR)
python datasets/hmm/generate_hmm_data.py --all --num_sequences 10000 --seq_length 256

# Generate all game datasets (RPS, Tic-Tac-Toe, Poker)
python datasets/games/generate_game_data.py --all --num_games 10000

# Generate Othello game transcripts
python datasets/othello/generate_othello_games.py --num_games 10000

# Generate synthetic HMMs with ground-truth belief trajectories
python datasets/synthetic/generate_synthetic_hmm.py --experiment all --num_sequences 5000 --seq_length 256
```

## 1. HMM Processes (`hmm/`)

**Source:** Based on Shai et al. (2024) "Transformers Represent Belief State
Geometry in their Residual Stream."

**Generator:** `hmm/generate_hmm_data.py`

### Mess3 Process

A 3-state HMM with vocabulary {A=0, B=1, C=2}.

- **Transitions:** Uniform over other states (no self-loops), p=0.5 each
- **Emissions:** Each state has a dominant token (p=0.9) with noise (p=0.05 each)
- **Belief geometry:** Lives on the 2-simplex (triangle). Produces fractal-like
  belief state geometry.

### Z1R (Zero-One-Random) Process

A 3-state HMM with binary vocabulary {0, 1}.

- **States:** Z (always emits 0), O (always emits 1), R (emits 0 or 1 with p=0.5)
- **Transitions:** Uniform over other states (no self-loops)
- **Key feature:** The R state is ambiguous, creating interesting belief dynamics.

### RRXOR (Random-Random-XOR) Process

A deterministic process with triplet structure.

- Tokens come in groups of 3: (a, b, a XOR b) where a,b ~ Bernoulli(0.5)
- Vocabulary: {0, 1}
- **Key feature:** The third token is deterministic given the first two. Has 36
  distinct belief states in a 4-simplex.

### Output Format

- `{process}_emissions.npy`: shape `(N, T)` -- observed tokens
- `{process}_states.npy`: shape `(N, T)` -- ground-truth hidden states
- `{process}_config.json`: process parameters (transition/emission matrices)

### Loading

```python
import numpy as np
emissions = np.load("datasets/hmm/mess3_emissions.npy")  # (num_seq, seq_len)
states = np.load("datasets/hmm/mess3_states.npy")        # (num_seq, seq_len)
```

The `GenericHMM` class in `generate_hmm_data.py` also provides:
- `belief_update(belief, observation)`: Single-step Bayesian update
- `compute_belief_trajectory(emissions)`: Full belief sequence

## 2. Simple Game Datasets (`games/`)

**Generator:** `games/generate_game_data.py`

### Rock-Paper-Scissors (RPS)

- **Vocabulary:** {R=0, P=1, S=2}
- **Sequence format:** [p1_move, p2_move, p1_move, p2_move, ...]
- **Strategies:** random, counter, win-stay-lose-shift, mixed
- **Output:** `rps_games.npy` shape `(num_games, 2 * rounds_per_game)`

### Tic-Tac-Toe

- **Vocabulary:** {0-8} for board positions (row-major)
- **Output:** `tictactoe_games.npy` (padded to length 9) + `tictactoe_outcomes.npy`
  (1=X wins, -1=O wins, 0=draw)
- **Strategies:** random or smart (heuristic)

### Simple Poker (Kuhn Poker)

- **Vocabulary:** {J=0, Q=1, K=2, check=3, bet=4, call=5, fold=6, p1_wins=7, p2_wins=8}
- **Format:** [card_p1, card_p2, action1, action2, ..., result]
- **Output:** `poker_hands.npy` (padded, shape varies by hand length)

### Loading

```python
import numpy as np
rps = np.load("datasets/games/rps_games.npy")  # (num_games, seq_len)
```

## 3. Othello Game Transcripts (`othello/`)

**Source:** Based on OthelloGPT (Li et al., ICLR 2023).

**Generator:** `othello/generate_othello_games.py`

Sequences of legal Othello moves generated by random play on an 8x8 board.
Each game is a sequence of integers in [0, 59] (60-token vocabulary, excluding
the 4 center squares that are pre-filled).

```bash
# Generate 10,000 games (~3.5 MB)
python datasets/othello/generate_othello_games.py --num_games 10000 --output othello_games.npy

# Generate with 64-token space (including center squares)
python datasets/othello/generate_othello_games.py --num_games 10000 --token_space 64
```

**Output:** NumPy array shape `(num_games, max_game_length)`, padded with -1.
Typical game length is exactly 60 moves.

**Key properties for residual stream analysis:**
- Rich hidden state (full 8x8 board with 3 states per cell)
- Sequential dependencies (legal moves depend on full board state)
- Prior work shows transformers learn world models (Li et al., 2023)

**External data sources:**
- Original OthelloGPT repo: https://github.com/likenneth/othello_world
  - Generates 20M+ games via `data/othello.py` with `ood_num=-1`
- Pre-trained weights: https://drive.google.com/drive/folders/1bpnwJnccpr9W-N_hzXSm59hT7Lij4HxZ
- HuggingFace: search "othello" (e.g., `taufeeque/othellogpt`)
- WTHOR database (championship games): https://www.ffothello.org/informatique/la-base-wthor/

## 4. Synthetic HMM Experiments (`synthetic/`)

**Generator:** `synthetic/generate_synthetic_hmm.py`

Controlled HMM experiments with **known ground-truth belief geometry**. In
addition to emissions and hidden states, these generators compute exact
**Bayesian belief trajectories** -- the posterior over hidden states after each
observation. This is the ground truth for evaluating whether transformer
residual streams learn to represent belief geometry.

### Available Experiments

| Experiment | States | Vocab | Description |
|-----------|--------|-------|-------------|
| `2state` | 2 | 2 | Binary HMM, noise=0.1. Belief = line segment. |
| `2state_clean` | 2 | 2 | Low noise (0.01). Beliefs cluster at vertices. |
| `2state_noisy` | 2 | 2 | High noise (0.3). Beliefs spread uniformly. |
| `3state` | 3 | 3 | Triangle belief simplex, noise=0.05. |
| `3state_noisy` | 3 | 3 | Noisier triangle (0.2). |
| `4state` | 4 | 4 | Tetrahedron belief simplex. |
| `5state` | 5 | 5 | 4-simplex belief geometry. |
| `asymmetric` | 2 | 2 | Asymmetric transitions (sticky + transient state). |
| `clock4` | 4 | 4 | Deterministic 4-cycle with noisy emissions. |
| `clock8` | 8 | 8 | Deterministic 8-cycle with noisy emissions. |

```bash
# Generate all experiments
python datasets/synthetic/generate_synthetic_hmm.py --experiment all --num_sequences 5000 --seq_length 256

# Generate a specific experiment
python datasets/synthetic/generate_synthetic_hmm.py --experiment 3state --num_sequences 10000 --seq_length 512
```

### Output Format

- `{name}_emissions.npy`: shape `(N, T)` -- observed tokens
- `{name}_states.npy`: shape `(N, T)` -- ground-truth hidden states
- `{name}_beliefs.npy`: shape `(N, T+1, K)` -- exact Bayesian belief trajectories
- `{name}_config.json`: full HMM specification (transition/emission matrices)

## Notes

- All generated data files (`.npy`, `.pkl`, `.csv`, etc.) are excluded via `.gitignore`
- All generators use seeded RNGs for reproducibility (default seed: 42)
- Generators are lightweight: 10K sequences take seconds to produce
- For large-scale experiments (>100K sequences), consider generating data
  on-the-fly during training rather than storing to disk
- For the research hypothesis, the key analysis is comparing residual stream
  geometry at early vs. late context positions (many-shot effect)
